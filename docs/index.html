    export-defaults

    $export-defaults: map-merge((

  // This property determines the unique status of exported item, essentially
  // changes state between class and id, where unique property set to false
  // equals to a class.

  'unique': false,

), if(global-variable-exists(export-defaults), $export-defaults, ()));

    Description

    Any property in this map is automatically added to every export item. Properties added via export or extend mixins take precedence over defaults.

    media

    @function media($value, $prop: width) { ... }

    Description

    Returns pixel value of media scale according to $value property. Use of media-between, media-only, media-up-from and media-up-to mixins is highly recommended instead of creating new media rules through this function. Media breakpoints are completely customizable through $const-media-scale constant. Note that both option on $prop is relevant only when media breakpoint is defined as map of width and height properties.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $value

    Scale or number

    		Number or String—none
    $prop

    Determines what value is returned, can be width, height or both

    		Stringwidth

    Example

    media(desktop) // returns 1680px

    mixins

    media-between

    @mixin media-between($lower, $upper, $prop: width, $operator: and, $direction: $const-media-direction) { ... }

    Description

    Wraps the content with a specified media query between $lower and $upper values. Use of media-up-from mixin is highly preferred before any other media mixin (not a requirement). The $operator parameter determines relationship between width and height properties, only relevant when $prop is set to both. Media breakpoints are completely customizable through $const-media-scale constant.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $lower

    		Number or String—none
    $upper

    Scale or number

    		Number or String—none
    $prop

    Determines what value is returned, can be width, height or both

    		Stringwidth
    $operator

    Determines relationship between width and height, can be and or or

    		Stringand
    $direction

    Determines the direction for media query, can be up or down

    		String$const-media-direction

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    @include media-between(phone, desktop) { ... }

    size

    @mixin size($size, $method: rem) { ... }

    Description

    Allows to use shorthand properties to represent width and height. Unitless values are handled as aspect ratio. Skip token false can be used to skip width or height property. This mixin already includes unit conversion, use non-relative units for input value, can be disabled by setting $method parameter to false. This mixin is alternative to postcss-short-size plugin.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $size

    Size, or space separated list of width and height (set false to skip)

    		Number or List—none
    $method

    Function called on the value, use em or rem

    		Bool or Stringrem

    Example

    @include size(640px 4/3)

    Throws

    • Height property (size) cannot be calculated, because width is skipped or defined as aspect ratio as well.

    • Width property (size) cannot be calculated, because height is skipped or defined as aspect ratio as well.
