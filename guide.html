      # Guide

      # Install

      You can use your favorite package manager with access to npm registry.

      # npm
npm install esm-scss

# yarn
yarn add esm-scss

      # Usage

      Everything that is needed is included in dist/esm file, simply import it and it's done.

      TIP

      This of course must correctly point to node_modules directory, but that's up to you to make sure it's correct.

      @import 'esm-scss/dist/esm';

      # Modules

      # Create Module

      Modules are created via export mixin.

      @include export(<module-name>, <props>);

      TIP

      This mixin will throw error when multiple modules with identical names are created. If you want to have modules with same names, use groups to recognize them between each other.

      For example, if I want to create a user module with background-color and size properties, the export mixin will look like this:

      @include export(
  user,
  (
    'background-color': gray,
    'width': 40px,
  )
);

      # Update Module

      Updating or adding new properties to module is done via extend mixin.

      @include extend(<module-name>, <props>);

      TIP

      This mixin will throw error when attempting to extend non-existing module.

      For example, if my user module has different background-color in another theme, I'll do the following:

      @include extend(
  user,
  (
    'background-color': blue,
  )
);

      # Use Module

      Modules are used via import mixin, while properties are extracted using get function.

      @include import(<module-name>);

      TIP

      The import mixin will automatically generate class from module name. Classes can be modified via template constants. If you don't want generated classes at all and use your own, use props mixin instead.

      For example, to use my user module:

      @include import(user) {
  background-color: get(background-color);
  height: get(size);
  width: get(size);
}

      My resulting user css would look like this:

      .user {
  background-color: blue;
  height: 40px;
  width: 40px;
}

      # Groups

      Groups can be used to separame module types (or for whatever reason you like, it's up to you). Groups first need to be created via export-groups constant, see constants. This constant is just a simple map, where map key is readable group representation, and map value is internal group id, also used for class generation.

      TIP

      Add $export-group: component; before the initial export mixin. All subsequent export, extend, import and props mixins will automatically select that group without dedicated group parameter.

      $export-group: component;
@include export(user, ( ... ));
@include import(user) { ... }

      For example, I want to have two module groups, component and design. First, I correctly add the group names and identifiers to the appropriate constant:

      $const-export-groups: (
  'component': 'c',
  'design': 'd',
);

      After that, I can create my user module with group as third parameter:

      @include export(
  user,
  (
    'background-color': gray,
    'width': 40px,
  ),
  component
);

      Then use my user module with group as second parameter:

      @include import(user, component) {
  background-color: get(background-color);
  height: get(size);
  width: get(size);
}

      My compiled css would look like this:

      .c-user {
  background-color: gray;
  height: 40px;
  width: 40px;
}

      # Export Defaults

      Map of default properties can be added to every created module. By default, only unique property is present, which determines whenever class or id is generated via import mixin, where unique: true will generate id instead of class.

      WARNING

      This variable must be placed before any export mixins.

      $export-defaults: (
  'unique': false,
);
